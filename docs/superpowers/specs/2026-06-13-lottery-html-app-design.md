# 彩票分析 HTML 应用 — 设计文档

日期:2026-06-13
状态:已确认,待实现

## 目标

把现有的 `lottery.py`(命令行脚本)改造成一个纯静态 HTML 应用。用户双击打开网页,
分双色球和大乐透两个板块,点击"生成"即可更新一个可浏览的分析表格,该表格可下载为
与现有 openpyxl 样式一致的 Excel,并新增基于频率加权算法的号码推荐功能。

**免责声明**:彩票开奖完全随机,任何算法都无法提高中奖概率。推荐功能仅供娱乐,页面需明确标注
"本推荐仅供娱乐参考,不构成任何投注建议"。

## 形态与文件结构

一个文件夹,本地双击即可运行,无需启动服务:

- `index.html` —— 主页面(HTML 结构 + CSS 样式)
- `lottery.js` —— 数据抓取、分析逻辑、表格渲染、推荐算法
- `exceljs.min.js` —— 本地 Excel 导出库(ExcelJS,MIT 许可,~280KB)

技术选型说明:
- **导出库用 ExcelJS**(非 SheetJS):SheetJS 免费版不支持单元格样式;ExcelJS 支持字体颜色、
  背景填充、边框、合并单元格、列宽,正好覆盖现有 openpyxl 的全部样式。
- **库放本地**:页面本身要联网抓数据,但库文件放本地保证页面框架永远能打开。

## 页面布局

顶部两个标签页(Tab)切换 **双色球** / **大乐透**。每个板块内含:

1. 【生成/更新】按钮 —— 点击后增量抓取数据 → 跑分析 → 渲染表格
2. 表格区域 —— 完整复刻 Excel 的多级表头、五行颜色、分区底色、边框
3. 【下载 Excel】按钮 —— 用 ExcelJS 导出 `.xlsx`,样式与页面一致
4. 【号码推荐】区域 —— 频率加权算法推荐一组号码,含【换一组】按钮
5. 顶部或推荐区附近的免责声明文字

## 数据流与缓存

**增量缓存(localStorage)**:

1. 点击"生成" → 读取 localStorage 中已存的历史数据
2. 查询本地最新期号 → 只 fetch 500.com 上新增的期数
3. 合并新数据并写回 localStorage
4. 取最近 180 期跑分析 → 同一份数据喂给三个消费者:HTML 表格渲染、ExcelJS 导出、推荐算法
5. **首次使用**(本地为空):抓取一批打底(最近约 300 期)

数据源(假定可正常访问,用户在别处已验证):
- 双色球:`http://datachart.500.com/ssq/history/newinc/history.php?start=...&end=...`
- 大乐透:`http://datachart.500.com/dlt/history/newinc/history.php?start=...&end=...`
- 解析 `tr.t_tr1` 行,提取期号、红球、蓝球;号码格式 `"1.2.3.4.5.6+7"`(`.` 分隔球号,`+` 分隔红/蓝区)

**不再保留**原脚本的全量历史 CSV 导出功能。

## 分析逻辑移植(lottery.py → lottery.js)

**关键约束**:输出列的数量和顺序必须与多级表头完全对应。这是原 Python 代码最易出错的点
(`multi_index` / `multi_index2` 的长度必须等于生成的列数)。

需移植的核心函数:

- **五行映射**(`get_element_type`):号码末位 → 土(0,5)/水(1,6)/火(2,7)/木(3,8)/金(4,9)
- **相克关系**(`get_prop_type`):相邻两球五行配对 → 刑 / ↓克 / ↑克 / ↓生 / ↑生。
  比较每球五行与其前一个球(用 '木' 作为首个哨兵);首行无前驱,置空(NaN)。
- **统计特征**:
  - 和值(红球求和)
  - 奇数 / 偶数计数
  - 分区计数:双色球 1-9 / 10-19 / 20-29 / 30-33;大乐透 1-9 / 10-19 / 20-29 / 30-35
  - 连号(`count_consecutive_numbers`:连续递增的段数)
  - 重号(`count_repeat_numbers`:与上一期重复的号码个数)
  - 同位(末位 `% 10` 重复的个数)
  - 余数(`% 3`)展开成宽表(`process_yushu`)

**双色球**:红球 6 个(1-33)+ 蓝球 1 个(1-16)。蓝球算五行/相克/余数。
**大乐透**:前区 5 个(1-35)+ 后区 2 个(1-12)。后区也要算完整的一套(五行、相克、和值、
奇偶、连号、重号、同位)。

两个板块各自的列结构参照 `lottery.py` 中 `shuangseqiu()` 和 `daletou()` 的输出,
以及 `multi_index` / `multi_index2` 的表头定义。

## 表格渲染与样式

**HTML 表格**:用两行 `<thead>` 实现多级表头(对应 pandas MultiIndex 的两层)。

**上色规则**(与现有 Excel 一致,用 CSS class 驱动):
- 相克文字颜色:↑克/↓克 → 红色(FF0000);刑 → 绿色(00FF00);↑生/↓生 → 蓝色(0000FF)
- 分区列底色:1-9 → 浅黄(FFFFE0);10-19 → 浅蓝(ADD8E6);20-29 → 浅绿(90EE90);
  30+ → 浅紫(E6E6FA)
- 边框:所有单元格细黑边框;文字居中

**Excel 导出**:用 ExcelJS 把上述 CSS class 映射成单元格的 fill / font / border;
合并单元格实现多级表头;列宽参照原 `column_dimensions` 设置。

## 号码推荐(频率加权算法)

基于历史窗口(最近 N 期,默认 100 期)为每个板块生成推荐:

**双色球**:
1. 统计窗口内每个红球(1-33)出现次数 → 频率权重
2. 以频率为权重做**不放回加权抽样**,抽 6 个红球(热号概率高,冷号仍有机会)
3. 蓝球(1-16)同理加权抽 1 个
4. 红球升序输出

**大乐透**:前区在 1-35 加权抽 5 个,后区在 1-12 加权抽 2 个,各自升序。

**轻量合理性校验**(已确认加入):
- 抽出后检查"和值"与"奇偶比"是否落在历史常见区间(例如历史和值的 10%~90% 分位之间)
- 若明显偏离则重抽(设最大重试次数,避免死循环)
- 目的:避免推出"全奇数""和值极端"等历史上几乎不出现的组合,让推荐更合理

**交互**:【换一组】按钮重新加权抽样,得到不同但合理的组合。

## 测试与验证

- 用一组已知号码手工核对五行、相克、连号、重号、同位等特征列是否与 Python 版一致
- 校验生成的列数与多级表头长度严格相等(渲染前断言)
- 导出的 .xlsx 用 Excel/openpyxl 打开,核对样式(颜色、表头、边框、列宽)
- 推荐算法:多次点击"换一组"应产生不同组合,且和值/奇偶落在合理区间
- 由于无法在此环境真实访问 500.com,数据抓取部分按用户"别处可跑通"的前提实现;
  分析与渲染逻辑用本地构造的样例数据验证
